Self-hosted · kubeadm · AWS EC2

Kubernetes Microservices on AWS

An end-to-end microservices deployment on a self-managed Kubernetes cluster (kubeadm) running on AWS EC2, provisioned with Terraform and shipped via GitHub Actions on a self-hosted runner. Built to exercise the fundamentals — not the managed abstractions — of cluster networking, DNS, IaC, and CI/CD.

Kubernetes kubeadm AWS EC2 Terraform Calico CNI CoreDNS GitHub Actions Self-hosted runner gRPC Flask Docker Buildx (multi-arch)

What this project demonstrates

kubeadm cluster setup & debugging
Multi-arch container builds (arm64 + amd64)
Cluster networking internals with Calico
DNS and service routing via CoreDNS
AWS Security Groups designed for k8s
Terraform state as source of truth
CI/CD with self-hosted GitHub runners
